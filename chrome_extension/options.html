Bridge Settings

Pick the task type being edited, set its bridge-control label, and choose the ChatGPT project IDs used by each account.

Task type project IDs

Each task type has two project IDs: the same project created in two ChatGPT accounts. Enter a raw 32-character ID, g-p- ID, or full project URL for each account. The selected account is opened when you switch task type from the bridge control menu.

Status Messages

Boilerplate Prompt

Prompt text and placeholder shortcuts for the selected task type.

Keyword Highlights

These rules are saved for the selected task type above. Matched strings are highlighted in ChatGPT messages. Adjacent terms are included before or after the match within the selected range. Normal strings ignore punctuation and hyphens; r- regex lines run against the raw text so symbols, commas, and decimals still work. Prefix a matched-string line with -- to exclude it from that rule.

Add one target per line. Prefix with r- for regex, !r- for priority regex, or -- to subtract a target. Use --r- for regex exclusions.

Add one adjacent term per line. Prefix with r- for regex, or !r- for priority regex when included.

Analysis TOC Buttons

These buttons are saved for the selected task type above. Edit each jump button's order, name, active color, screen side, scroll offset, and case sensitivity. Column placement and opacity are in the UI tab.

Custom TOC targets

Custom string buttons are saved for the selected task type and can be added alongside built-in buttons. Put one detection string per line; repeated matches cycle within the latest assistant response. Matching is case-insensitive unless the button's Case toggle is enabled.
