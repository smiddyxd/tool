# Downloadable Utilities Manual — Final

This manual contains original task guidance plus added supplemental guidance. Added guidance is placed next to the most relevant official passage and is meant to clarify practical application, especially for screenshot/OCR-based rating workflows.

## Source labels

- **Official guidance** = original task instructions and category guidance.
- **Added guidance** = supplemental practical guidance added to clarify rating behavior.
- **Non-manual indicators** = useful case-specific indicators not explicitly listed in this manual; these may be mentioned in per-task output when relevant.

# Official guidance

In this project you are asked to review ad creatives or web pages, and to decide whether or not they fall under the Downloadable Utilities category. The results of this work will allow publishers to exclude ads related to Downloadable Utilities from their websites.

## Task structure

Each task will contain one ad creative or one web page. Your task is to review this ad creative or web page and decide whether it falls under the Downloadable Utilities category.

1. **IMPORTANT: When labelling a web page, please take its URL into account.** You will find the URL right underneath the web page. If there is evidence in the text of the URL itself that the content of the webpage belongs to the Downloadable Utilities category, please rate accordingly. Example:
    - [http://www.disk-tools.com/download]

### Added guidance — URL evidence

A visible webpage URL can be sufficient evidence on its own when it clearly points to a utility download, even if the visible page or creative is sparse.

2. **Some ad creatives and web pages might be unrateable.** The most common reasons to select "unrateable" are:
    - content is not primarily in your rating language (german or english)
    - content does not load
    - content has no text, or contains too little content to understand what it is about
    - content does not render well enough to interpret
    - there's an error preventing you from viewing the content
    - page contains little content and requires registering or logging in
    - web page contains a video and lacks textual content that describes its contents
    - web page has offensive content that you're not comfortable examining closely
    
    Please leave a brief comment explaining why you think the ad creative or the web page is Unrateable.
    
3. **Comments**. If you have comments regarding particular tasks, please use the comment section for that.
4. **Not sure how to rate.** If you are not sure how to rate a whole task or part of it, please still provide your best answer and additionally add a brief comment saying that you are not sure. For example, "not sure how to rate" or "not sure how to rate creative since ...". It is very important that you leave comments in such situations. It will allow us to confirm the ratings and enrich this set of guidelines with additional explanations.

### Added guidance — screenshot and OCR evidence discipline

Use only visible evidence from the screenshot or OCR text: visible text, visible URL, imagery, product name, CTA, page state, and visible branding. Do not assume what happens after clicking. Do not infer hidden landing-page behavior. If evidence is unclear, mark it unclear rather than inventing details.

### Added guidance — Unrateable vs uncertain

- Unrateable means you cannot assess the content at all because it failed to load, is in the wrong language, is behind a login wall, and so on.
- Uncertain means there is assessable content but classification is difficult; in that case, give your best rating and leave a comment.
- Do not use Unrateable as a fallback for ambiguous downloadable-utilities cases.

# Scope of the Downloadable Utilities category

## Official guidance

The Downloadable Utilities category includes ad creatives and web pages related to software for download which is designed to enhance the functionality of an operating system or device, whether desktop or mobile. Typical examples include anti-virus software, file converters, driver updaters, system cleaners, download managers, disk defragmenters, codecs, browser toolbars, ringtones, screensavers, wallpapers, and so on. Ads often claim to increase performance, enhance security, or otherwise fix common ailments, and sometimes imply that something is wrong with the user’s device as a way to encourage clicks. Such software is often offered for free, but need not be. Also includes mobile subscription services which most typically promise ringtones (or humorous videos) in exchange for a recurring fee billed through the phone service provider.

### Added guidance — practical core test

Ask whether the content involves downloadable software that enhances, fixes, protects, or customizes a device. This is a shorthand for applying the official scope above, not a replacement for it.

## Official guidance

It excludes ad creatives and web pages related to online media streaming services such as Netflix or Vudu, general mobile apps or games, and web browsers are not in scope. "Casting" apps, which work with existing streaming apps to send programs to other devices (such as TVs), are also not in scope..

Ads and pages which are positive for this category sometimes have certain features, such as those listed below. (However, note that ads and pages in scope for this category need not have any of them.)

- they sometimes state or imply that something is wrong with the user's device
- they sometimes use eye-catching graphical tricks to gain the user's attention, often with the intention of alarming the user, e.g., flashing, blinking, appearing to vibrate
- they sometimes state or seem to imply that they are official offers from Microsoft, Apple, or other operating system makers, or official compenents of common operating systems such as Android or Windows, when they are not
- ads sometimes have a prominent "download" button or link which suggests that you can immediately download the item rather than going to a landing page, and in some cases these are deceptively presented in such a way as to suggest that they are part of the page they appear in
- ads sometimes have a button or link which suggests that by clicking a certain operation will be performed, when in fact they take the user to a landing page; an example would be a button or link labeled "Clean Now" for a system cleaner
- they sometimes offer direct download subscriptions, especially for mobile devices, often with missing or obscured information about billing and rates (these are more common in some markets and rare in others)

### Added guidance — applying positive features

- The word “download” alone is not enough. Ask what is being downloaded and whether it is utility software or an included mobile subscription service.
- Red flags confirm the category but do not define it. A clean, calm, professional utility ad can still be in scope if it offers downloadable utility software.
- Progress bars, scan animations, warning panels, or diagnostic visuals that imply something is already happening on the device are strong positive clues, especially when paired with device-problem language such as “clean,” “scan,” “virus,” “errors,” “update,” or “fix.”
- Unofficial Microsoft, Apple, Android, Windows, or trademark-heavy branding is a strong positive clue when affiliation is unclear, especially when combined with update, repair, driver, security, or system-performance claims.

## Official guidance

If any of the above traits are present, it is highly likely that the ad or page is in scope.

# Topics included in the Downloadable Utilities category

## Official guidance

### **(1) Content related to downloadable utilities**.

This includes software for download which is designed to enhance the functionality of an operating system or device, whether desktop or mobile.

**Positives**
• anti-virus and malware software
• system cleaners
• file converters
• driver updaters
• download managers
• disk defragmenters
• anything which offers to speed up a device, its internet connection or its wireless connections
• anything which offers to remove unwanted things from a device
• screensavers, wallpapers
• mobile OS launchers

### Added guidance — practical positive signals for visible-evidence rating

• security tools more broadly, including antivirus, malware protection, privacy/security shields, and other protective utilities
• speed boosters, RAM optimizers, battery savers, storage cleaners, cache cleaners, and performance-tuning tools
• themes, icon packs, wallpapers, screensavers, and mobile launchers when tied to downloadable customization software
• archive/unzip tools, duplicate-file finders, password-recovery tools, PDF/file compressors, and similar device/file-management utilities
• utility keywords in product names, URLs, or domains, such as cleaner, booster, optimizer, updater, converter, defrag, codec, repair, driver, shield, or antivirus, when supported by device/software context
• device/OS terms such as PC, Mac, Android, RAM, battery, storage, cache, registry, malware, Wi-Fi, errors, or drivers, when paired with download/install/scan/clean/fix/update context
• installer/package clues such as `.exe`, `.msi`, `.apk`, “install,” “setup,” or “download for Windows/Android,” when the product is utility-like
• visible app-store category or label such as Tools, Utilities, Security, or Cleaner, when the product context supports a utility reading

## Official guidance

**Negatives**
• online, mobile or desktop games
• web browsers
• productivity apps such as: word processors, spreadsheets, calendars
• shopping apps such as Amazon Shopping
• messaging apps such as Twitter
• actual operating systems such as Windows, Android, or OS X (but be careful, some ads pretend to be official when they are not -- if uncertain, assume the ad is in scope)
• offers for instructional materials regarding operating systems or apps of various kinds
• apps which work with existing streaming apps (such as Netflix, Hulu, HBOGo) to allow streaming programs to be played in other devices such as TVs

### Added guidance — practical negative signals for visible-evidence rating

• no software is involved at all, unless the mobile subscription-service subsection applies
• the focus is entertainment or content rather than device functionality, unless the mobile subscription-service subsection applies
• “download” refers only to an ordinary document, form, coupon, ticket, brochure, PDF, media file, or other non-software item
• the offer is a general game, media app, browser, productivity app, shopping app, messaging app, streaming service, or casting app already excluded by official guidance
• the product is a web-only or SaaS-style tool with no downloadable software and no device-enhancement function
• the visible content points to lead generation, surveys, sweepstakes, insurance, finance, physical goods, or other non-software/content offers

### Added guidance — ambiguous “download application form” outlier

A “download application form” style example should be treated cautiously. By the main category definition, an ordinary downloadable form or document is not a downloadable utility, because it is not software that enhances, fixes, protects, or customizes a device.

However, because a similar canonical/example creative appears to be treated as positive, sufficiently similar cases may be rated Yes with a brief uncertainty comment.

Do not generalize this outlier. If the task is materially dissimilar from that example, return to the main manual rule: the content should involve downloadable utility software or an included carrier-billed mobile subscription service. Ordinary forms, PDFs, job/service applications, coupons, brochures, or paperwork downloads should normally be No unless there is additional evidence tying them to downloadable software.

# Mobile subscription services

## Official guidance

### **(2) Content related to mobile subscription services**.

This includes services which offer ringtones, humorous videos, celebrity gossip, etc., in exchange for a recurring fee billed through the user's phone service provider. Note that ads will sometimes end with an amount in a currency you may not be familiar with.

### Added guidance — mobile subscription signals

Additional positive signals:
- For mobile subscription services, recurring phone-provider billing is what puts the content in scope, even when the content itself is entertainment-oriented.
- Prices marked `/wk`, `/week`, `/mo`, `/month`, or similar can be strong clues when paired with mobile/carrier billing context.
- Ringtones, short videos, celebrity content, humorous videos, gossip, and similar offers can still be in scope when billed through the phone provider.

Additional negative signals:
- Streaming services or content subscriptions that are not billed through the phone provider remain out of scope.
- Mobile entertainment/content offers without visible recurring carrier-billing context should not be treated as positive under this subsection based on content alone.

# Task analysis template

This section is the per-task analysis template the LLM should execute after reading the manual above. It structures the rating output.

## Task-analysis process

1. Identify what is visibly offered using only screenshot/OCR evidence.
2. Check whether the task is assessable at all. If not, rate Unrateable and briefly explain why.
3. If the task is assessable but classification is difficult, give the best Yes/No rating and add a brief uncertainty comment.
4. Do not use Unrateable as a fallback for ambiguous but assessable Downloadable Utilities cases.
5. Sparse creatives can still be rateable if product name, visible URL, CTA, imagery, or branding provides enough evidence.
6. If an exception or outlier example seems relevant, apply it only to sufficiently similar cases and return to the main category definition otherwise.

## Per-task indicator extraction

For each task, list only indicators that actually apply to the visible content. Separate matching indicators by source:
- Official = directly present in official guidance.
- Added = present in the added guidance sections of this manual.
- Non-manual = useful practical indicators not explicitly present in the manual; these may support reasoning but must not override the manual.

Use “None” in any indicator-table cell or Unrateable-indicator line where no matching indicator applies.

## Before-finalizing check

Before finalizing, check:
- Am I using Unrateable only because the task cannot be assessed at all?
- If I rated Yes, is there visible evidence of downloadable utility software or an included carrier-billed mobile subscription service?
- If I rated No, am I ignoring strong visible URL/product/CTA evidence of a utility?
- If an exception seems relevant, is this task sufficiently similar to the canonical/example exception?
